with vision.GestureRecognizer.create_from_options(options) as recognizer:
The with statement is Python's way of saying "set this up, and automatically clean it up when I'm done,
 even if something crashes." The recognizer loads a machine learning model into memory and may open system resources. 
 Using with guarantees those get released properly when the loop exits — you don't have to manually close anything.


Mediapipe: 
result = recognizer.recognize(mp_image)
This is where MediaPipe actually does its work. You pass in one frame (wrapped as an mp.Image object), 
and it runs the model and returns a result object containing:

result.hand_landmarks — a list of 21 landmark positions per detected hand
result.gestures — a list of ranked gesture guesses per hand (e.g. "Thumb_Up" at 94% confidence)
result.handedness — whether each hand is left or right
- if no hands detected then return None(s) and loops don't run


for i, hand_landmarks in enumerate(result.hand_landmarks):
    draw_landmarks(frame, hand_landmarks)

    if result.gestures and i < len(result.gestures):
        top = result.gestures[i][0]
        draw_gesture_label(frame, top.category_name, top.score, i)

the if loop is to ensure that only 1 gesture is being recorder per hand
